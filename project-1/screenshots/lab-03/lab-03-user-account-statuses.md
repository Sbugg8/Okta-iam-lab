# Lab 03 – Manage User Account Statuses in Okta

## What is this?
In this lab, I managed the full range of user account status transitions 
in Okta — covering password reset, password expiration, account lockout, 
suspension, and deactivation.

## Why does it matter?
These are the most common IAM admin tasks in any enterprise environment. 
Help desk tickets, security incidents, and employee offboarding all run 
through these exact workflows. Knowing when to reset vs expire vs suspend 
vs deactivate is a core IAM skill — and it's tested on the Okta 
Certified Professional exam.

## What I learned
- **Suspend** = reversible, immediate access block. Used for investigations, 
leave of absence, or pending offboarding.
- **Deactivate** = end state for offboarding. Account preserved for audit 
trail but fully disabled.
- Every status change is logged in the Okta System Log — critical for 
compliance and SOC 2 auditing.
- The default lockout policy triggers after 10 failed attempts — admins 
must manually unlock unless self-service is configured.
- Temporary passwords force the user to set a new credential on next 
login — used when a compromise is suspected.
